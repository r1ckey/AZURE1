Azure Container Instances (ACI)

  • サーバーレス: サーバーのプロビジョニングや管理が一切不要。秒単位の課金。
  • 素早い起動: 単一のコンテナを数秒で起動。一時的なバースト処理に最適。

ACI は、最も手軽にコンテナを実行できるサービスです。Kubernetes のような複雑なオーケストレーションは不要で、「とにかく今すぐ、このコンテナイメージを動かしたい」という時に力を発揮します。データのバッチ処理や、CI/CD パイプラインの一時的な実行環境として非常に強力です。

Azure Container Registry (ACR)

  • プライベート管理: 自分の組織専用のコンテナイメージ保存庫（レジストリ）。
  • イメージスキャン: 保存されたイメージの脆弱性を自動でチェック（Defender 連携）。
  • 作成・配信: リージョンを跨いでイメージをレプリケーションし、AKS への反映を高速化。

ACR は、開発したコンテナをパッケージ化した「イメージ」を「安全に保管」する場所です。Docker Hub のプライベート版だと考えると理解しやすいでしょう。Azure コンピューティングサービス（AKS や App Service）と密に連携し、セキュアなデプロイを実現します。

Azure Kubernetes Service (AKS)

  • オーケストレータ: 多数のコンテナ（Pod）の配置、死活監視、負荷分散を自動化。
  • コントロールプレーン提供: 複雑な Kubernetes マスターの管理を Azure が無料（一部例外）で行う。
  • 統合: VNet、Azure 認証、監視機能とワンクリックでシームレスに統合。

AKS は、エンタープライズ向けの強力なコンテナ管理プラットフォームです。

AKS のスケーリング機能

  • HPA (Horizontal Pod Autoscaler): コンテナの台数を負荷に応じて増減。
  • クラスターオートスケーラー: Pod が溢れたとき、自動でノード（サーバー）を追加。
  • バーチャルノード: 台数不足のとき、一時的に ACI を使ってサーバーレスに拡張。

AKS の最大の魅力は「二段階のスケーリング」です。まずコンテナを増やし、もし物理的なサーバーリソースが足りなくなれば、自動的に Azure 仮想マシンが追加されます。これにより、急激なアクセス増に対してもシステムを停止させることなく対応できます。
