Azure Load Balancer (L4)

  • レイヤー 4: TCP/UDP レベルでの負荷分散。きわめて高速。
  • ハッシュベース: IP アドレスやポートの 5 タプルで分散先を決定。
  • パブリック / 内部: インターネット公開用と、VNet 内の内部用を選択可能。

Azure で最もシンプルかつ高速なロードバランサーです。パケットの中身（URLなど）を見ずに、送り先（バックエンド VM 群）へ振り分けるため、低レイテンシーが求められる非 HTTP 通信やサーバー間の負荷分散に最適です。Standard SKU を使えば、可用性ゾーン (AZ) 間の負荷分散もサポートされます。

Azure Application Gateway (L7)

  • レイヤー 7: HTTP/HTTPS 用の高度な分散。URL パスやホスト名での振り分け。
  • WAF 統合: ウェブアプリケーションファイアウォールによる不正アクセス遮断。
  • セッション維持: 特定のクッキーにより同じユーザーを同じサーバーへ。

Web サイトの運用に特化したバランサーです。URL パスに基づくルーティングや WAF による保護が可能です。

Azure Front Door (グローバル配信)

  • グローバル L7: 世界中のエッジロケーションからコンテンツを高速配信。
  • エニーキャスト: 最も近い Azure ネットワーク入り口へユーザーを誘導。
  • 統合防御: DDoS 防御（Shield）やグローバル WAF と密に連携。

Front Door は「Web サイトの入り口を全世界に広げる」サービスです。単なるバランサーだけでなく CDN（配信ネットワーク）の機能も兼ね備えており、世界中のどこからアクセスしても、爆速で Web アプリを提供できます。リージョンを跨いだマルチリージョン構成の入り口として最適です。

Azure Traffic Manager (DNS ベース)

  • DNS ルーティング: クエリ結果（回答する IP）を地理的条件などで変更。
  • 重み付け / 優先度: 50%:50% で分けたり、メインが死んだらサブへ切り替える。

Traffic Manager は、通信そのものを中継するのではなく、「次はこの IP へ行ってね」と名前解決の結果を切り替えるサービスです。そのため非 HTTP プロトコルを含めた全通信に対応でき、グローバルな可用性を高める際の交通整理として利用されます。
