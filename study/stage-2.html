Azure Virtual Machines (VM)

  • IaaS: OS レベルの管理が可能。カスタマイズ性が最も高い。
  • VM サイズ: A, D (汎用), E (メモリ), F (計算) など用途別に選択。
  • スポット VM: 未使用のリソースを大幅割引で購入（中断のリスクあり）。

Azure VM は、クラウド上の「仮想サーバー」です。Linux や Windows などの OS を選び、CPU やメモリのサイズを自由に変更できます。Azure の VM サイズには名前のルールがあり、例えば D-シリーズ はバランスの取れた汎用向け、E-シリーズ は大容量メモリを必要とする分析などに向いています。

高可用性設計 (可用性セット / 可用性ゾーン)

  • 障害ドメイン (Fault Domain): 物理的なラック（電源やスイッチ）を共有する単位。
  • 更新ドメイン (Update Domain): 同時に再起動される単位。パッチ適用時の停止を防ぐ。
  • 可用性ゾーン (AZ): さらに広い範囲（建物単位）での冗長化。

可用性セット (Availability Sets) を使用すると、複数の VM が同じ物理的な「ラック」に配置されないよう調整できます。

Virtual Machine Scale Sets (VMSS)

  • スケーリング: 負荷に応じて VM の台数を自動で増減（オートスケール）。
  • 一貫性: 同じ構成の VM を大量に、素早く展開。

VMSS は、同一構成の VM 群を「一つのグループ」として管理する仕組みです。トラフィックが急増した時に自動的にサーバーを追加し、負荷が下がれば削減することで、コストの無駄を省きつつ安定したサービスを提供できます。

Azure App Service (PaaS)

  • PaaS: サーバー管理は完全に Azure 側。コードを置くだけで Web サイトが動く。
  • スロット: 本番とステージング環境をワンクリックで入れ替え（スワップ）。
  • マネージドセキュリティ: 証明書の管理や脆弱性対応も Azure がサポート。

App Service は、Web アプリケーション、API、モバイルのバックエンドに最適な完全マネージドなプラットフォームです。サーバー管理が一切不要で、開発者はプログラムコードの開発に集中できます。デプロイスロット機能を使えば、新しいバージョンを裏側でテストし、問題がなければ一瞬で本番と切り替えることが可能です。
