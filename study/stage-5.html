Azure SQL Database (PaaS)

  • マネージド: OS やデータベースのパッチ適用は全て Azure が自動で行う。
  • スケーリング: vCore モデルや DTU モデルを選択可能。
  • 弾力性プール (Elastic Pool): 複数のデータベースでリソースを共有してコスト削減。

Azure SQL Database は、高い互換性と信頼性を備えたマネージドなリレーショナルデータベースです。特に Business Critical 層 を選べば、非常に高い IOPS と低いレイテンシー、そして高速なリカバリが保証されます。また、多数の小規模な DB を並行稼働させるなら 弾力性プール でリソースをまとめ、個別のピークを相殺して費用を抑える設計が可能です。

自動フェイルオーバーとリージョン地理レプリケーション

  • アクティブ地理レプリケーション: 最大 4 つのセカンダリリージョンへデータを非同期コピー。
  • オートフェイルオーバーグループ: 読み取り用・書き込み用のエンドポイントを自動で切り替え。

万が一リージョン規模の障害が発生しても、あらかじめ オートフェイルオーバーグループ を組んでおけば可用性を維持できます。

Azure Cosmos DB (NoSQL)

  • グローバル分散: 世界中にデータをボタン一つで配信可能。
  • マルチモデル: SQL API、MongoDB、Cassandra などの API をサポート。
  • 超高速な応答: 読み取り・書き込みともに 1 桁ミリ秒以内の低遅延。

Cosmos DB は、現代的なWebアプリやモバイル、ゲーム開発に最適な分散型データベースです。スキーマレスなデータを保存でき、何万件ものトラフィックを瞬時に処理する高いスケーラビリティが最大の特徴です。さらに、世界中のどこからでも低レイテンシーでデータにアクセスできるよう、複数リージョンでの同時書き込み（マルチマスター）にも対応しています。

5 つの一貫性レベル

  • 1. 強固 (Strong): 全ての読み取りで直前の書き込みが反映されることを保証。
  • 2. セッション (Session): 同一ユーザー（セッション）内での読み取り一貫性を保証。既定。
  • 3. 最終的 (Eventual): いずれどこかで同期されるが、一時的に古いデータが見える可能性がある。最速。

Cosmos DB の設計で最も面白いのが、一貫性と速度（パフォーマンス）のトレードオフ を 5 つのレベルから選べる点です。厳密なデータの一致が必要な銀行系アプリなら「強固」、一般的な SNS 等なら「セッション」など、用途に合わせて柔軟にエンジニアリングできます。
