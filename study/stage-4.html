ストレージアカウント

  • Blob ストレージ: オブジェクト（ファイル）保存。
  • Azure Files: SMB/NFS 互換のファイル共有（ファイルサーバー）。
  • Azure Queues / Tables: メッセージや簡易データの保存。

Azure のストレージサービスの全ての入り口となるのが ストレージアカウント です。非構造化データ（画像やバックアップ）の保存に最適な Blob や、既存のファイルサーバーをクラウド化するための Azure Files など、多岐にわたる用途を一括管理します。

データの冗長性 (Redundancy)

  • LRS (ローカル冗長): 単一のデータセンター内で 3 つコピー。最も安価。
  • ZRS (ゾーン冗長): リージョン内の 3 つの AZ に分散。AZ 単位の障害に耐える。
  • GRS (ジオ冗長): リージョンペア（遠隔地）に自動コピー。大規模災害に備える。

Azure では、ストレージアカウント作成時に 冗長性レベル を選択します。

Blob アクセス層とライフサイクル管理

  • ホット層: 頻繁にアクセスするデータ。保存コスト高、転送コスト低。
  • クール層/クール(Cold)層: たまにアクセスするデータ。保存コストは抑えられる。
  • アーカイブ層: ほぼアクセスしない保存用。最安だが、取り出し（リハイドレート）に数時間。
  • ライフサイクル管理: 日数経過で自動的に層を移動。

データの「鮮度」に合わせて保存場所を最適化することで、コストを大幅に削減できます。さらに「作成から 30 日経過したファイルは自動的にクール層へ」といった自動化を ライフサイクル管理 で設定することが推奨されます。

Azure Files

  • 共有ストレージ: SMB または NFS プロトコルで、VM もオンプレミスも同時接続可能。
  • Azure File Sync: オンプレミスの物理ファイルサーバーの中身を Azure と同期・拡張。

Windows ファイルサーバーをクラウドに移行する際に最も便利なサービスです。既存のネットワークドライブ設定をそのまま利用できるため、アプリケーションの改修なしでクラウド化を推進できます。
