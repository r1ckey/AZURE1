Azure Monitor

  • メトリクス: CPU、メモリ、ディスク、ネットワークのパフォーマンス数値。
  • ログ: 各リソースのアクティビティや構成変更、OS 内のログ。
  • アラート: 数値が閾値を超えたらメールや Webhook で通知。

Azure Monitor は、統合的な監視のハブです。全てのリソースからメトリクスとログを収集します。

Log Analytics (KQL)

  • 一元管理: 複数の VM やサービスから大量のログを一つのワークスペースに集約。
  • KQL: Kusto Query Language による強力な検索と分析。

膨大なログの中から「エラーが発生した時間帯の不審な操作」を見つけ出す際に活躍します。SQL に似た KQL という独自言語を使用して、数百万件のログから瞬時に結果を抽出・グラフ化できます。インシデント調査の強力な味方です。

Azure Backup

  • Recovery Services Vault: バックアップデータを安全に保管する「金庫」。
  • ファイルレベル復元: VM 全体だけでなく、中の特定のファイルだけを書き戻す。
  • 長期保管: セキュリティコンプライアンス等で必要な数年単位の保存。

誤操作によるデータの消失や、不審なランサムウェアによるファイル破壊からシステムを守るための唯一の砦です。Azure VM、SQL DB、Azure Files などのバックアップ設定を、エージェントを最小限に抑えながらシンプルに自動化できます。

Azure Site Recovery (ASR)

  • 災害対策 (DR): リージョン規模の障害時に、他リージョンの VM を起動（レプリケーション）。
  • RTO / RPO: 目標復旧時間 (RTO) と目標復旧時点 (RPO) を最小限に抑えた継続性を実現。

ASR は単なるバックアップではなく「予備機への瞬時の切り替え」を行うサービスです。プライマリリージョンが壊れても、設定済みのセカンダリリージョンで数分以内にビジネスを再開させる、高度な可用性設計の中核を担います。
