Azure Functions (FaaS)

  • サーバーレス: インフラ管理不要。コードをアップロードするだけで特定のトリガーで動作。
  • 従量課金プラン: 実行回数と実行時間のみ課金。アイドルコストはゼロ。
  • 汎用性: HTTP リクエスト、タイマー、DB の更新などをきっかけに動作。

Azure Functions は、現代の「サーバーを意識させない」開発の主軸です。Web API のエンドポイントを即座に作成したり、ストレージにファイルがアップロードされた瞬間に画像処理（リサイズ等）を実行したりと、非常に高い柔軟性を誇ります。

Azure Logic Apps

  • ローコード: 視覚的なデザイナーで、多数のサービス間を繋ぐワークフローを作成。
  • コネクタ: Office 365, Slack, Salesforce など、Azure 以外の外部サービスとも容易に連携。

複雑なプログラムコードを書かなくても、「特定のメールが届いたらデータを抽出し、SQL に保存して Slack に通知する」といったビジネスオートメーションが可能です。Azure の各種サービスを「糊（のり）」のように繋げる、非常に強力な統合ツールです。

Azure Event Grid

  • イベントルーティング: 「リソースが作成された」といったイベントをリアルタイムに検知し、適切な処理先へ通知。
  • 疎結合: 送信側と受信側を直接繋がない、柔軟な設計（Event-driven）を支える。

Event Grid は、Azure における「イベント」の交通整理役です。疎結合なアーキテクチャを支えます。

Azure Service Bus (メッセージング)

  • エンタープライズ統合: 信頼性の高いメッセージキュー（順番待ち）を提供。
  • 疎結合の極み: 受信側が一時的に忙しくても、キューにメッセージを溜めておくことでデータを喪失しない。

Service Bus は、システム間の通信を「非同期」に行うための重厚なメッセージングサービスです。数百万というメッセージを順番に、かつ確実に処理するための FIFO（先入れ先出し）や、特定の条件（トピック）に合致した受信側だけにデータを送る高度なフィルタリングもサポートします。
